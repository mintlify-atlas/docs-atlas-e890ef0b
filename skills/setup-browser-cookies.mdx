## What it does

`/setup-browser-cookies` is your **session manager mode**. Before `/qa` or `/browse` can test authenticated pages, they need cookies. Instead of manually logging in through the headless browser every time, this skill imports your real sessions directly from your daily browser.

It auto-detects installed Chromium browsers (Comet, Chrome, Arc, Brave, Edge), decrypts cookies via the macOS Keychain, and loads them into the Playwright session.

<Note>
An interactive picker UI lets you choose exactly which domains to import - no cookie values are ever displayed.
</Note>

## When to use it

Use `/setup-browser-cookies` when:
- You want to test authenticated pages with `/qa` or `/browse`
- You need to import login sessions from your real browser
- You want to avoid manually logging in through the headless browser every time
- You're testing staging or production environments that require auth

## How it works

<Steps>
  <Step title="Find the browse binary">
    The skill checks for the browse binary and runs setup if needed.
  </Step>
  
  <Step title="Open the cookie picker">
    ```bash
    $B cookie-import-browser
    ```
    
    This auto-detects installed Chromium browsers and opens an interactive picker UI in your default browser.
  </Step>
  
  <Step title="Select domains">
    In the picker UI:
    - Switch between installed browsers
    - Search domains
    - Click "+" to import a domain's cookies
    - Click trash to remove imported cookies
  </Step>
  
  <Step title="Cookies are loaded">
    After you confirm, cookies are decrypted and loaded into the Playwright session.
    They persist between `/browse` commands.
  </Step>
</Steps>

## Cookie picker UI

The picker is served on the same port as the browse server (no extra process). It provides:

- **Browser switcher** - Toggle between installed browsers (Comet, Chrome, Arc, Brave, Edge)
- **Domain search** - Filter domains by name
- **Import/remove** - Click "+" to import, trash to remove
- **Privacy** - Only domain names and cookie counts shown, no cookie values

<Warning>
First import per browser triggers a macOS Keychain prompt - click "Allow" or "Always Allow".
</Warning>

## Direct import

If you know the domain and browser, skip the UI:

```bash
$B cookie-import-browser <browser> --domain <domain>
```

Supported browsers:
- `comet`
- `chrome`
- `arc`
- `brave`
- `edge`

Example:
```bash
$B cookie-import-browser arc --domain staging.myapp.com
```

## How cookies persist

The browse session persists cookies between commands. Once imported:

1. Cookies work immediately in the next `/browse` or `/qa` command
2. They stay active until the browser session shuts down (30 min idle timeout)
3. You don't need to re-import for each test

If the session shuts down, re-run `/setup-browser-cookies` to restore them.

## Verify imported cookies

After import:

```bash
$B cookies
```

Shows all cookies in the current session. You'll see a summary of imported domains.

## Security notes

<Warning>
- **Real sessions** - Imported cookies give full access to the accounts you're logged into
- **Use staging, not production** - Unless you intend to test production
- **Session timeout** - The headless browser shuts down after 30 min idle
- **Cookie visibility** - The picker UI shows domain names and counts, but never cookie values
- **Keychain access** - First import per browser requires macOS Keychain approval
</Warning>

## Supported browsers

The skill auto-detects installed Chromium-based browsers:

| Browser | Auto-detected |
|---------|---------------|
| Comet | ✓ |
| Chrome | ✓ |
| Arc | ✓ |
| Brave | ✓ |
| Edge | ✓ |

Firefox and Safari are not supported (different cookie storage).

## Tips

<Accordion title="Which browser to use?">
Use whichever browser you're normally logged into. The skill auto-detects all installed Chromium browsers - pick the one with your active sessions.
</Accordion>

<Accordion title="What if cookies don't work?">
Verify they imported:
```bash
$B cookies
```

If missing, re-run `/setup-browser-cookies`. The session may have timed out.
</Accordion>

<Accordion title="Can I import multiple domains?">
Yes. In the picker UI, click "+" for each domain. Or run the skill multiple times with different domains:

```bash
You:   /setup-browser-cookies github.com
You:   /setup-browser-cookies staging.myapp.com
```
</Accordion>
