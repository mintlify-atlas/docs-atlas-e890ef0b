This quickstart gets you from zero to your first working workflow. You'll install gstack, run a basic browser command, and experience the power of having specialized AI modes at your fingertips.

## Prerequisites

Before you begin, ensure you have:

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) installed and running
- [Git](https://git-scm.com/) installed
- [Bun](https://bun.sh/) v1.0 or higher installed

<Note>
If you don't have Bun installed, run: `curl -fsSL https://bun.sh/install | bash`
</Note>

## Step 1: Install gstack

Open Claude Code and paste this command. Claude will handle the installation:

```markdown
Install gstack: run `git clone https://github.com/garrytan/gstack.git ~/.claude/skills/gstack && cd ~/.claude/skills/gstack && ./setup` then add a "gstack" section to CLAUDE.md that says to use the /browse skill from gstack for all web browsing, never use mcp__claude-in-chrome__* tools, and lists the available skills: /plan-ceo-review, /plan-eng-review, /review, /ship, /browse, /qa, /setup-browser-cookies, /retro. Then ask the user if they also want to add gstack to the current project so teammates get it.
```

<Steps>
  <Step title="Clone and build">
    The setup script will:
    - Clone the repository to `~/.claude/skills/gstack`
    - Install dependencies with Bun
    - Compile the browser binary (~58MB)
    - Create symlinks for all skills
  </Step>
  
  <Step title="Verify installation">
    Claude will update your CLAUDE.md file and confirm the skills are available.
  </Step>
</Steps>

<Info>
Installation takes about 30 seconds. The compiled browser binary works on macOS and Linux (x64 and arm64).
</Info>

## Step 2: Try Your First Workflow

Let's build a simple feature using multiple cognitive modes.

### Plan Mode: CEO Review

Start by describing what you want to build. Let Claude help you think bigger:

```
I want to add a photo upload feature to my listing app.

/plan-ceo-review
```

Claude will pressure-test whether "photo upload" is the real feature, or if the real job is helping sellers create listings that actually sell. It might suggest:
- Auto-identify the product from the photo
- Pull specs and pricing comps from the web
- Draft the title and description automatically
- Suggest the best hero image

### Engineering Mode: Lock in Architecture

Once you've decided on the product direction, shift to engineering mode:

```
/plan-eng-review
```

Claude will provide:
- Architecture diagrams
- Data flow and state machines
- Failure modes and edge cases
- Test coverage matrix

### Build the Feature

Exit plan mode and start implementing. Claude will code the feature based on the technical plan.

### Review Mode: Find Structural Bugs

Before shipping, run a paranoid review:

```
/review
```

Claude will look for:
- Race conditions
- N+1 queries
- Trust boundaries
- Missing indexes
- Bad retry logic

### Ship It

Once the review is clean, ship in one command:

```
/ship
```

This will:
- Sync with main
- Run tests
- Resolve any Greptile reviews
- Push the branch
- Create a PR

## Step 3: Test with Browser Automation

Now let's test your feature with the headless browser.

### Basic Navigation

```
You: /browse

Claude: Browser session started.

You: browse goto http://localhost:3000
```

### Get Interactive Elements

Use snapshots to see what you can interact with:

```
You: browse snapshot -i

Claude: [Returns accessibility tree with @e1, @e2, @e3 refs]

You: browse click @e5
     browse fill @e2 "test@example.com"
     browse screenshot /tmp/form.png
```

### QA Testing

Run a full QA pass on your feature:

```
/qa
```

Claude will:
1. Analyze your git diff
2. Identify affected routes
3. Test each page automatically
4. Generate a health score report

<Tip>
For authenticated testing, first run `/setup-browser-cookies` to import your real browser sessions. Then `/qa` can test pages behind login.
</Tip>

## Step 4: Run a Retrospective

At the end of the week, analyze your shipping velocity:

```
/retro
```

You'll get:
- Commits, LOC, and test ratio
- Peak coding hours
- Biggest ships of the week
- Team member breakdown with praise and growth opportunities
- Shipping streaks

## Common Workflows

<AccordionGroup>
  <Accordion title="Feature Development (Plan → Code → Review → Ship → QA)">
    The complete cycle:
    1. `/plan-ceo-review` - Validate the product direction
    2. `/plan-eng-review` - Lock in architecture
    3. Code the feature
    4. `/review` - Find structural bugs
    5. `/ship` - Automated release workflow
    6. `/qa` - Test everything that changed
  </Accordion>
  
  <Accordion title="Bug Fix (Review → Ship)">
    For simple fixes:
    1. Fix the bug
    2. `/review` - Make sure you didn't introduce new issues
    3. `/ship` - Push and create PR
  </Accordion>
  
  <Accordion title="QA on Staging">
    Test a deployed environment:
    1. `/setup-browser-cookies staging.myapp.com`
    2. `/qa https://staging.myapp.com --quick` - 30-second smoke test
    3. Or `/qa https://staging.myapp.com` - Full systematic test
  </Accordion>
</AccordionGroup>

## Troubleshooting

<Accordion title="Skills not showing up?">
  Run `cd ~/.claude/skills/gstack && ./setup` to rebuild symlinks.
</Accordion>

<Accordion title="Browser binary not found?">
  Run `cd ~/.claude/skills/gstack && bun install && bun run build` to compile the browser.
</Accordion>
